# Где используется БЭМ в проекте "Шахматы"

## 1. Блоки — основные строительные единицы

В проекте выделены следующие независимые блоки:

- **Блок `board`** — вся шахматная доска вместе с обрамлением
- **Блок `status`** — панель с информацией о ходе и кнопкой
- **Блок `info-panel`** — верхняя декоративная панель с символами
- **Блок `footer`** — нижняя часть с информацией об используемых технологиях

Каждый блок визуально и функционально независим. Их можно перемещать, копировать или изменять, не затрагивая другие части страницы.

---

## 2. Элементы — составные части блоков

Внутри блоков выделены более мелкие компоненты:

### В блоке `board`:
- **Элемент «сетка»** — контейнер для клеток
- **Элемент «ячейка»** — каждая отдельная клетка доски

### В блоке `status`:
- **Элемент «иконка хода»** — отображает символ текущего игрока
- **Элемент «сообщение»** — текст "Ход белых/чёрных"
- **Элемент «кнопка сброса»** — для начала новой игры

### В блоке `info-panel`:
- **Элемент «нотация»** — набор шахматных символов
- **Элемент «бейдж»** — декоративная плашка с надписью

### В блоке `footer`:
- **Элемент «орнамент»** — декоративные символы по бокам

---

## 3. Модификаторы — изменения внешнего вида и состояния

Для элементов используются модификаторы, которые меняют их отображение:

### Для элемента «ячейка»:
- **Модификатор цвета** — определяет, клетка светлая или тёмная
- **Модификатор выделения** — показывает, какая фигура выбрана
- **Модификатор подсветки** — показывает возможные ходы для выбранной фигуры

> **Важно**: модификаторы никогда не используются сами по себе, они всегда дополняют базовый элемент.

---

## 4. Как именуются классы в БЭМ

В проекте строго соблюдается соглашение об именовании:

- **Блок** — просто одно слово (`board`, `status`, `footer`)
- **Элемент** — блок + два подчёркивания + элемент (`board__cell`, `status__message`)
- **Модификатор** — элемент + одно подчёркивание + значение (`board__cell_selected`, `board__cell_color_white`)

Это делает код самодокументируемым: даже без комментариев понятно, что к чему относится.

---

## 5. Почему это именно БЭМ, а не просто CSS-классы

### Отсутствие вложенности
В проекте нет селекторов типа `.board .status button`. Каждый элемент стилизуется через свой собственный класс, что исключает конфликты и зависимости.

### Независимость блоков
Блок `board` ничего не знает о блоке `status`, и наоборот. Их можно менять независимо.

### Переиспользуемость
Если бы потребовалось разместить на странице две доски (например, для демонстрации), достаточно просто скопировать блок `board` — стили не сломаются.

### Понятность для новых разработчиков
Глядя на класс `status__reset-btn`, сразу ясно: это кнопка сброса, которая находится внутри блока статуса.

---

## 6. Как БЭМ взаимодействует с JavaScript

JavaScript в проекте тоже учитывает БЭМ-структуру:

- При выборе фигуры скрипт добавляет ячейке модификатор `selected`
- При расчёте возможных ходов добавляется модификатор `highlight`
- При смене игрока обновляется текст в элементе `status__message`

Таким образом, даже динамические изменения соответствуют БЭМ-методологии.

---

## 7. Преимущества БЭМ в этом конкретном проекте

- **Масштабируемость** — легко добавить новые блоки (например, «захваченные фигуры» или «таймер»)
- **Поддержка** — можно изменять стили одного блока, не боясь сломать другие
- **Читаемость** — структура страницы понятна на уровне HTML
- **Совместная работа** — разные разработчики могут работать над разными блоками без конфликтов

## 8. Примеры БЭМ-структуры в проекте

## Полная спецификация классов

### Блок `board`
| Элемент | Модификаторы | Описание |
|---------|--------------|----------|
| `board__grid` | - | Контейнер-сетка 8x8 |
| `board__cell` | `board__cell_color_white` | Светлая клетка |
| `board__cell` | `board__cell_color_black` | Тёмная клетка |
| `board__cell` | `board__cell_selected` | Выбранная фигура |
| `board__cell` | `board__cell_highlight` | Возможный ход |

### Блок `status`
| Элемент | Описание |
|---------|----------|
| `status__turn` | Контейнер для иконки и сообщения |
| `status__turn-icon` | Иконка текущего игрока |
| `status__message` | Текстовый индикатор хода |
| `status__reset-btn` | Кнопка новой игры |

### Блок `info-panel`
| Элемент | Описание |
|---------|----------|
| `info-panel__notation` | Набор шахматных символов |
| `info-panel__badge` | Декоративный бейдж |

### Блок `footer`
| Элемент | Описание |
|---------|----------|
| `footer__ornament` | Декоративные элементы по бокам |

## Соглашение об именовании

- **Блок**: `board`, `status`, `info-panel`, `footer`
- **Элемент**: `board__cell`, `status__message`, `footer__ornament`
- **Модификатор**: `board__cell_selected`, `board__cell_color_white`

> Двойное подчёркивание (`__`) отделяет элемент от блока.  
> Одинарное подчёркивание (`_`) отделяет модификатор от элемента.
